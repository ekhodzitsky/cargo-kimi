## Cross-Cutting Concerns

### 1. `PUB_FN_RE` regex is duplicated and incomplete everywhere

The pattern `^\s*(pub\s+)?(async\s+)?(unsafe\s+)?fn\s+` appears in `contracts.rs`, `fix.rs`, and `verify.rs` — each with the same gap: `const fn`, `extern fn`, `pub(crate) fn` are invisible. A single shared, correct regex would fix C-6, F-3, and improve V-1 simultaneously.

### 2. Test-block brace counting is broken in the same way in two modules

Both `contracts.rs` (C-1) and `fix.rs` (F-4) use `trimmed.matches('{').count()` instead of the project's own `count_braces_outside_strings()`. The function exists but is never called for test-block tracking.

### 3. Security-critical `util.rs` has zero tests

The path validation function is the project's only security boundary (called from MCP server). It has a confirmed symlink bypass (U-1), possible null-byte bypass (U-2), and exactly 0 test cases.

### 4. Dead config pattern: parsed → stored → never read

`score.ignore` (CF-1) and `fail_on_drop` (CF-4) are deserialized from `.kimi.toml` but never consumed. Users who configure them are silently misled. This suggests features were planned but never wired up.

### 5. Unused `#[allow(dead_code)]` newtype structs at file bottoms

`LspUri`, `McpMethod`, `CommandName`, `Score`, `Strictness` — pattern structs that exist in 5+ modules, all marked `#[allow(dead_code)]`, never used. Likely code-generation artifacts.

### 6. Eight modules have zero test coverage

`mcp.rs`, `verify.rs`, `watch.rs`, `workspace.rs`, `util.rs`, `config.rs`, `lib.rs`, and `trend.rs` (unit) — collectively ~900 LOC with no dedicated tests. Includes the security-critical path validation and the public API dispatch layer.
